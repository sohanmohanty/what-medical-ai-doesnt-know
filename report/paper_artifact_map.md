# Paper Artifact Map

This file maps the paper's claims to the saved artifacts in the repository. The main benchmark claims are supported by the canonical `paper_core` run, while focused extension analyses are supported by retained comparison outputs.

## Canonical paper-core path

Command:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_paper_core.ps1
```

Primary outputs:

- `results/metrics/paper_core_metrics.csv`
- `results/metrics/paper_core_summary.csv`
- `results/predictions/paper_core_predictions.csv`
- `results/manifests/paper_core_run_manifest.json`
- `results/manifests/paper_core_run_spec.md`
- `figures/runs/paper_core/`
- tracked paper-facing figures in `figures/paper/`
- frontend artifact: `artifacts/frontend/paper_core_explorer.json`

This path is intended to support the paper's core benchmark story:

- clean vs corrupted benchmark evaluation
- WDBC vs Statlog robustness contrast
- logistic regression vs random forest vs gradient boosting
- MCAR vs MAR vs MNAR trends across rates

## Companion Explorer

The interactive web app uses `artifacts/frontend/paper_core_explorer.json`, generated by `scripts/export_frontend_artifacts.py`, to present the saved benchmark results. The explorer is a communication layer, not a separate source of experimental claims.

## Focused Extension Analyses

Some focused analyses in the paper remain tied to separately generated comparison outputs:

- calibration-specific reliability work
- focused imputer comparisons
- training-regime comparisons
- missingness-indicator comparisons

These remain valuable and are retained for provenance, but they are not yet fully unified into the canonical `paper_core` path.

## Interpretation

The repo should currently be read as:

- one canonical official benchmark path for the main comparative study
- retained extension outputs for focused analyses that have not yet been fully unified
- a companion frontend artifact for interpretation and exploration
