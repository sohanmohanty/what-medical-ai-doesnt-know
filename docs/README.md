# Documentation Guide

The `docs/` folder contains project documentation, not private notes from the build process.

Start here:

- `project-brief.md`: the purpose, scope, and central question of the project
- `methodology.md`: datasets, missingness mechanisms, models, metrics, and limitations
- `architecture.md`: how the Python benchmark, JSON artifact, and web app fit together
- `migration-plan.md`: completed work, current limitations, and future roadmap
- `content-strategy.md`: writing and explanation guidelines for the website
- `next-steps.md`: short list of practical follow-up tasks

The main `README.md` is the front door for the repository. These documents provide more detail for readers who want to understand the design choices behind the benchmark and the interface.
