# Technical Documentation

The `docs/` folder contains supporting technical documentation for the benchmark and web interface.

Start here:

- `methodology.md`: datasets, missingness mechanisms, models, metrics, and limitations
- `architecture.md`: how the Python benchmark, JSON artifact, and web app fit together
- `deployment.md`: static deployment configuration and artifact flow

The main `README.md` is the entry point for the repository. These files provide additional detail for reproducing, inspecting, and deploying the project.
